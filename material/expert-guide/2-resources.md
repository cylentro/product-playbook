## Documentation & Visualization

Standardizing how we communicate requirements and logic ensures clarity during handovers and reduces technical debt.

### Markdown Standards

Essential for writing clean PRDs and documentation in tools like GitHub, Azure DevOps, and Notion.

* [Markdown](https://www.markdownguide.org/): A comprehensive reference for syntax.
* [Markdownlint](https://github.com/DavidAnson/markdownlint): Tooling to ensure your documentation follows best-practice styles.

### Mermaid Diagrams (Visual Logic)

Use Mermaid to create diagrams within your Markdown files (User Flows, Business Logic, System Sequences).

* [Mermaid Diagram](https://mermaid.dev/): Documentation and syntax guide.
* [Mermaid Live Editor](https://mermaid.live/): A sandbox to build and test diagrams before embedding them in your PRDs.
